[ Manager Login / Shift Overview
Manager opens the app → Sees list of active/open shifts (or waiters currently working).
Manager can start a new shift for a waiter who arrives (select waiter → set start time, optionally planned end time).

During the Day
All orders placed by a waiter are automatically linked to their current open shift.
The system always knows “which shift is responsible for this order”.

Closing Shifts (Clôture)
Manager goes to “Close Shifts” section.
Sees list of currently open shifts (sorted by waiter).
Manager closes them one by one:
Select a shift → Review summary (orders, total, cash, tips, etc.).
Option to add manual adjustments (voids, discounts, cash tip declaration…).
Confirm close → System records end_datetime and finalizes the numbers.

How to Handle Midnight-Crossing Shifts
This workflow solves your original problem naturally:

A waiter who started yesterday at 19:00 still has an open shift today at 01:00.
When the manager closes that shift at 03:30, the system pulls all orders from 19:00 yesterday until 03:30 today and attributes them to that single shift.
The next shift for that waiter (or another) starts fresh.

Key rule: An order always belongs to the active shift of the waiter at the moment the order was created.
